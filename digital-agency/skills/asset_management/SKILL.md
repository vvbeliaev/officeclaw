# Asset Management

## Структура папки клиента

```
clients/{chat_id}/
├── brief.md              # бриф проекта
├── assets/               # всё что пришло от клиента или собрано
│   ├── logo/             # логотип в разных форматах
│   ├── photos/           # фотографии
│   ├── fonts/            # шрифты если предоставил клиент
│   ├── moodboard/        # скриншоты Pinterest и других источников
│   │   └── palette.md    # собранные цвета HEX
│   └── texts/            # тексты от клиента (TXT, DOC)
└── artifacts/            # то что создаём мы
    ├── site/             # код сайта
    ├── *.pdf             # PDF материалы
    ├── *.png             # скриншоты превью
    └── phase_*_summary.yaml  # итоги фаз
```

## Именование файлов

- Нижний регистр, слова через underscore: `hero_photo.jpg`
- Без пробелов и спецсимволов
- Дата для версий: `brief_v2_2026-03-25.md`
- Скриншоты референсов: `ref_01.png`, `ref_02.png`

## Когда получаешь файл от клиента

1. Сохрани в соответствующую подпапку `assets/`
2. Если изображение > 2MB — запомни что нужна оптимизация перед использованием на сайте
3. Если текстовый контент — сохрани в `assets/texts/`, потом используй при написании копирайта

## Работа с изображениями

```bash
# Конвертация в WebP (если есть imagemagick)
convert input.jpg -quality 80 output.webp

# Проверка размера
ls -lh assets/photos/
```

Целевые размеры для сайта:

- Hero изображение: ≤ 300KB
- Gallery/секции: ≤ 150KB каждое
- Иконки/logos: SVG предпочтительно
