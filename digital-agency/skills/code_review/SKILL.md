# Code Review — Local Site Files

Адаптировано из Claude code-review skill. Применяется к локальным файлам сайта перед деплоем.

## Процесс

### Шаг 1. Определи scope

Прочитай файлы сайта: `index.html`, `style.css`, `script.js` (и другие если есть).
Не трать время на файлы которые не влияют на результат (README, конфиги).

### Шаг 2. Параллельный review по перспективам

Запусти 4 независимых анализа:

**А. Баги и логика**
- Сломанные ссылки и пути к ресурсам
- Незакрытые теги, невалидный HTML
- JS ошибки: undefined переменные, неправильные обработчики событий
- Form submission без action/handler

**Б. Мобильная адаптация**
- Viewport meta tag присутствует
- Нет элементов с фиксированной шириной > 375px без media query
- Touch targets ≥ 44x44px
- Текст читается без горизонтального скролла на 375px
- Images масштабируются корректно

**В. Производительность**
- Изображения без оптимизации (>500KB без reason)
- Render-blocking resources в `<head>` без `defer`/`async`
- Неиспользуемые CSS классы или большие инлайн-стили
- Нет кэширующих заголовков (не в scope HTML, но флагануть)

**Г. Доступность и SEO**
- `<title>` и meta description заполнены
- OG теги для социальных сетей
- Alt тексты для всех изображений
- Логическая иерархия заголовков (h1→h2→h3)
- Контраст: текст/фон достаточный

### Шаг 3. Оценка уверенности

Для каждой найденной проблемы оцени от 0 до 100:

- **0–25**: False positive, скорее всего не проблема
- **50**: Реальная проблема, но minor
- **75**: Реальная проблема, влияет на UX или SEO
- **100**: Критическая проблема, сломает функционал или мобилку

**Фильтр**: репортуй только проблемы с оценкой ≥ 75.

### Шаг 4. Fix или report

- **Оценка 100** → исправь немедленно, не жди апрува
- **Оценка 75** → исправь если просто, иначе запроси апрув с описанием
- **Оценка < 75** → не репортуй

## Что НЕ является проблемой

- Pre-existing issues (было до твоих изменений)
- Стилистические предпочтения без явного стандарта
- Issues которые prettier/linter поймает при сборке
- Отсутствие features которые не были в requirements

## Формат результата

```
CODE REVIEW: {количество} issues найдено

CRITICAL (100):
- [описание] → [файл:строка]

HIGH (75):
- [описание] → [файл:строка]

Исправлено автоматически: [список]
Требует апрува: [список если есть]
```

Если проблем нет:
```
CODE REVIEW: Чисто. Готово к деплою.
```
